<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| service worker → cached app shell | vite-plugin-pwa's generated SW precaches and serves the app shell; a stale SW could serve outdated code after deploy. |
| build tooling → shipped PWA assets | @tailwindcss/vite and vite-plugin-pwa generate CSS, manifest, and SW at build time; no runtime server exists (PLAT-04, fully client-side). |
| localStorage → app runtime | Settings written by GameContext (01-01) are read back on reload; untrusted browser-writable input crosses here (mitigated in 01-01, T-01-01). |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-01 | Tampering | Stale service worker serving outdated app shell | low | mitigate | registerType 'autoUpdate' (Pattern 4) activates new SW and reloads rather than trapping users on a cached old version. Availability/correctness, not a classic security breach. |
| T-02-02 | Spoofing | PWA manifest identity (name/icons/theme_color) | low | accept | Manifest identity is public, non-sensitive branding for a party game with no accounts (PLAT-04). No install-integrity control beyond HTTPS-served static assets is warranted. Accepted. |
| T-02-03 | Information Disclosure | Precached assets in the SW cache | low | accept | Only public static build output (JS/CSS/icons) is precached; no secrets or user data enter the cache. Accepted. |
| T-02-SC | Tampering | npm/pip/cargo installs | high | mitigate | No new package installs in this plan — vite-plugin-pwa and @tailwindcss/vite are already pinned and legitimacy-audited in RESEARCH.md (workbox-window is explicitly NOT installed per Pitfall 3). No [ASSUMED]/[SUS] install checkpoint required. |
</threat_model>
