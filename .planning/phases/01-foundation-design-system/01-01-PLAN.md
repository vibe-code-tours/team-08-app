<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage → app runtime | The user's own browser (or a browser extension / devtools) can write arbitrary bytes under the settings key; this untrusted input crosses into the app on every load via loadSettings(). |
| build tooling → shipped assets | @tailwindcss/vite and (in plan 01-02) vite-plugin-pwa transform source into shipped CSS/SW at build time; no runtime server trust boundary exists (PLAT-04, fully client-side). |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-01 | Tampering | loadSettings() JSON.parse on localStorage | medium | mitigate | Wrap JSON.parse/localStorage access in try/catch; malformed or tampered content falls back to defaultSettings and never throws (Task 3 acceptance criteria; RESEARCH Security Domain V5). |
| T-01-02 | Denial of Service | localStorage quota / private-browsing on saveSettings() | low | mitigate | saveSettings wraps setItem in try/catch and fails silently; in-memory state continues to work if storage is unavailable. |
| T-01-03 | Information Disclosure | Data persisted to localStorage | low | accept | Only non-sensitive GameSettings (difficulty/pack/timer toggle) are persisted; no PII, secrets, or auth tokens exist in this app (PLAT-04). Accepted. |
| T-01-SC | Tampering | npm/pip/cargo installs | high | mitigate | No new package installs in this plan — all deps (tailwindcss, @tailwindcss/vite, react) are already pinned in package.json and legitimacy-audited in RESEARCH.md (SUS too-new verdicts cross-verified as false positives). No [ASSUMED]/[SUS] install blocking checkpoint required. |
</threat_model>
