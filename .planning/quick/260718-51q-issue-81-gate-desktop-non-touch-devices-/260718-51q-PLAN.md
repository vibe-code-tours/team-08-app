<objective>
Gate desktop / non-touch devices with a QR-to-mobile handoff screen (GitHub issue #81).

Add feature-detection-based touch capability detection in a small testable hook, a dedicated Myanmar-language gate screen showing a QR code of the current URL plus a "Continue anyway" escape hatch, and wire it into App.tsx so the gate fires ONLY on touch-required game screens (never on intro/setup). Detection approach documented in CLAUDE.md so future sessions don't reinvent it.

Purpose: Desktop/laptop users currently hit a broken finger-placement UI with no explanation. This gives them a clear "switch to your phone" path while keeping setup/intro browsable and honoring the touch-only design in ADR 0001 (no mouse fallback).
Output: `useTouchCapability` hook + test, `DesktopGateScreen`, App.tsx wiring, `qrcode` dependency, CLAUDE.md note.
</objective>
