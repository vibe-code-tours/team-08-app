<objective>
Fix PWA update/cache staleness so new GitHub Pages deploys reach users on the next fresh
navigation instead of only after every tab is closed.

Purpose: The current `VitePWA({ registerType: 'autoUpdate' })` has no `workbox` block and no
`useRegisterSW`/`virtual:pwa-register` call, so a new service worker sits in the `waiting` state
until all tabs close. Because the deploy target is GitHub Pages, we cannot set Cache-Control
headers server-side, so the entire fix lives at the service-worker layer.

Output: A `workbox` block (skipWaiting + clientsClaim + NetworkFirst navigation caching),
a tap-to-reload `UpdateToast` component mounted in `App.tsx`, a CLAUDE.md learning bullet,
and ADR 0002 documenting the strategy.
</objective>
