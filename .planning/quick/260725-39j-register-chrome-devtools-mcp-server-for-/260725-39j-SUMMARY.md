## Outcome Update (2026-07-25, post-review)

`/review` on PR #102 (README overhaul) flagged committing `.mcp.json` as scope creep and a supply-chain concern: it auto-launches `npx -y chrome-devtools-mcp@latest` (unpinned) for every contributor with Claude Code open in this repo, unrelated to that PR's doc-only purpose.

Attempted to split it into a standalone PR (`chore/register-chrome-devtools-mcp`, commit `e96ef91`, cherry-picked from `9e14913`), but PR creation repeatedly failed with a persistent `500` from GitHub's API specific to this repo (confirmed via `gh`, raw REST `curl`, and the GitHub MCP tool; GitHub's status page showed all-clear).

**Final decision:** rather than keep retrying or merging an unreviewed shared config change, `.mcp.json` was reverted to a **local-only, gitignored file** — added `.mcp.json` to `.gitignore`, removed it from git tracking (commit `853d23b` on the docs PR branch), and recreated it untracked on disk so the chrome-devtools MCP server still works in this checkout. The `chore/register-chrome-devtools-mcp` branch (local and remote) was deleted since it's no longer needed.

Net effect: this quick task's original goal (chrome-devtools MCP working in this local checkout) still holds, but `.mcp.json` is no longer shared via git — each developer who wants it now creates their own local copy (see snippet in this plan's `<context>` block above) rather than it being provisioned repo-wide.
